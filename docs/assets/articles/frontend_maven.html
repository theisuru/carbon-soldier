Spring Angular integration with Maven frontend plugin

Normally you would pass run build as a configuration argument to npm goal as a last step of your maven front end plugin. Instead if you run build --prod it will build your angular application in production mode as if you ran `ng build --prod`. You can keep two profiles for default build and production build to easily integrate this in your CI/CD pipeline. Furthermore, you can keep multiple profiles for multiple environments if you are willing do modifications to your `angular.json` file and add new files to `$PROJECT/src/environments` directory. user `mvn clean install -Dprod` to build the production code
